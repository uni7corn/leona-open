# Changelog

All notable changes to Leona Android SDK are documented here.
Format follows [Keep a Changelog](https://keepachangelog.com/en/1.1.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [Unreleased]

### Added
- v0.4 Android evidence and privacy boundary documentation for the commercial
  pilot track. The SDK contract remains evidence-only: Android apps receive
  BoxIds, diagnostics, and redacted support material; customer backends own
  all final business decisions.
- v0.4 release notes draft covering public artifacts, readiness gates, external
  blockers, privacy rules, and explicit non-goals.
- v0.4 Android/Server aggregate release-readiness gate that composes the clean
  OEM ledger, Android matrix readiness, Maven Central readiness, backend wrapper
  mock HTTP smoke, public archive dry-run, public archive consumer smoke, and
  optional heavier Gradle/consumption gates.
- v0.4 Android publish workflow dry-run that verifies tag-time GitHub Release
  AAR/sha256 and GitHub Packages wiring without creating a tag or publishing
  artifacts.
- v0.4 release candidate manifest that composes readiness, archive consumer,
  publish workflow dry-run, public commit scope, and public release batch
  summaries for local pre-tag review without tagging, publishing, executing
  `git add`, staging files, starting paid devices, or printing secrets; strict
  pre-tag mode fails while version-alignment blockers remain.
- v0.4 release evidence pack generated by the release candidate manifest. It
  indexes readiness, archive consumer, publish workflow dry-run, public commit
  scope, and public release batch component summaries with paths, SHA-256
  digests, byte counts, before/after git index snapshots, and a redaction scan
  that also covers the generated evidence pack Markdown/JSON itself.
- v0.4 release evidence pack schema gate that validates the generated JSON
  fields, exact component set, referenced file SHA-256/byte counts, git index
  snapshots, and Markdown/JSON parity without treating synthetic
  `redactionScan=failed` as a schema failure.
- Synthetic generated-pack self-test for the release candidate redaction scan
  via `LEONA_RC_SELF_SCAN_TEST=1`.
- v0.4 tag release runbook that documents version alignment, annotated tag
  push, tag workflow verification, and post-release consumption smoke.
- v0.4 post-release consumption wrapper that defaults to a blocked summary
  before publication and can be made strict after the tag workflow publishes
  GitHub Release and GitHub Packages artifacts.
- v0.4 Android version marker gate that checks `VERSION_NAME`, SDK runtime
  constants, sample versionName, and release scripts before the final tag.
- v0.4 Android version bump plan gate that previews required runtime marker
  edits, optional sample/docs preview edits, replacement counts, and planned
  SHA-256 digests without writing files before the final release commit.
- v0.4 Android version bump dry-run gate that applies the planned public
  Android marker replacements in memory and verifies required markers converge
  to the target release version without writing files.
- v0.4 public commit scope gate that fails staged non-public paths and reports
  mixed iOS/Web/server/homepage/private/internal-doc work before a public
  Android GitHub commit.
- v0.4 public release batch planner that emits public candidate paths,
  do-not-stage paths, staged forbidden paths, and a non-executed stage-command
  draft for mixed workspaces; the generated draft is syntax-checked, parsed
  back into a path list, hashed, compared against the public batch list, and
  checked against do-not-stage paths before it is reported. The planner also
  runs the draft in `LEONA_ANDROID_STAGE_DRY_RUN=1` mode and verifies the git
  index is unchanged after `git add --dry-run`.
- v0.4 public release batch schema gate that independently validates generated
  batch summaries for path-list parity, SHA-256 digest correctness,
  do-not-stage exclusion, forbidden public prefixes, and dry-run git index
  preservation.
- v0.4 release candidate manifest schema gate that independently validates the
  generated RC manifest summary fields, component paths, no-tag/no-publish/
  no-stage guarantees, evidence pack schema status, public batch fields, git
  index preservation, and version-blocker semantics.
- v0.4 release candidate review wrapper that runs the manifest and schema gate
  together, writes one combined summary, and preserves the underlying failure
  behavior for strict pre-tag or synthetic-negative runs.
- v0.4 release candidate review schema gate that validates the wrapper summary,
  manifest/schema summary paths, no-tag/no-publish/no-stage/no-device
  guarantees, and failure semantics.
- v0.4 release candidate final review wrapper that runs the review wrapper and
  review schema gate together for one local pre-tag review command.
- v0.4 release candidate final review schema gate that validates the final
  wrapper summary, nested review/schema summary paths, no-tag/no-publish/
  no-stage/no-device guarantees, and failure semantics.
- Maven Central readiness gate that validates publication metadata and required
  environment variable names without printing secrets.
- Backend wrapper contract plus Node.js and Java server-side wrapper skeletons
  for signed evidence lookup, support bundle export, and feedback submission.

### Validation notes
- The v0.4 public-safe readiness gate currently passes local checks and reports
  remaining external blockers without embedding sensitive material.
- Public archive consumer smoke extracts the generated Android archive, checks
  required public files, rejects private/server/iOS/Web/homepage roots and
  local generated files, and syntax-checks archived shell scripts.
- Publish workflow dry-run verifies tag trigger, release/package permissions,
  versioned asset naming, prerelease tag handling, GitHub Packages publication,
  and absence of Central/provider/customer secret references in the public
  workflow.
- Release candidate manifest records component summary paths, archive digest,
  expected publish assets, public commit scope status, public release batch
  counts, do-not-stage counts, staged forbidden path counts, strict pre-tag
  mode, release evidence pack paths, release evidence pack redaction status,
  release evidence pack schema status, git index preservation status, and
  explicit external blockers in one local review artifact.
- Release candidate manifest schema gate validates the generated summary
  itself after normal, strict, or synthetic-negative RC runs, without staging,
  tagging, publishing, starting devices, or printing secrets.
- Release candidate review wrapper records both the manifest summary and schema
  summary paths in one local artifact for tag review.
- Post-release consumption wrapper records whether the real v0.4 artifact path
  has passed, is still blocked before publication, or failed under strict mode.
- Public commit scope gate records staged forbidden paths, public candidate
  paths, non-public dirty paths, and secret/raw-identifier scan status without
  printing secret values.
- Public release batch planner records the public Android batch and
  do-not-stage paths without executing `git add`, records the generated
  stage-command draft syntax result, records SHA-256 digests for the reviewed
  path lists, verifies the draft paths match the public batch, and verifies
  the draft does not contain do-not-stage paths. It additionally runs a
  `git add --dry-run` stage rehearsal and checks that the git index is
  preserved.
- Backend wrapper skeleton checks cover request signing, mock HTTP paths,
  redaction, packaging dry-runs, and absence of business allow/reject/block
  decisions.
- Version bump plan checks record planned `0.4.0` marker updates and remain
  read-only until the final release bump commit.
- Version bump dry-run checks prove the planned marker replacements converge in
  memory before any release bump commit is written.

## [0.3.0] - 2026-05-18

### Added
- Android API 23-30 validation coverage for the public SDK release gate,
  including API-level diagnostics that keep client behavior evidence-only.
- Cloud-phone evidence collection and validation records for hosted public
  reporting on Samsung/Asus sample devices.
- HMA, Magisk, Zygisk, and LSPosed provenance validation records for authorized
  test environments. These remain evidence signals; the Android SDK does not
  make allow/reject/block decisions.
- Attestation provider dry-run release gate for the sample Play Integrity
  bridge template, keeping real provider material outside the public
  repository.
- v0.3 release-readiness wrapper for the public SDK, local Gradle gate,
  clean-OEM ledger gate, and post-tag artifact verification checklist.

### Fixed
- Restored Android 6 / API 23 public hosted reporting compatibility for the
  Leona hosted endpoint by falling back to a bundled ISRG Root X1 trust anchor
  only after the platform trust manager fails for `leona.xiyanshan.com`.
  Custom hosts are not covered by this fallback, and configured certificate
  pins remain enforced.

### Validation notes
- API 23 hosted-reporting smoke passed with a real hosted AppKey after the
  trust fallback, returning a BoxId and canonical id without TLS trust-anchor,
  SSL handshake, auth, timestamp-skew, or server-5xx errors.
- API 24, 26, 29, and 30 emulator/AVD validation passed the local public SDK
  evidence collection flow.
- API 28 WeTest smoke and cloud-phone sample validation completed for the
  release boundary available before publishing.
- Real custom ROM/GSI/unlocked-device samples, broader external emulator
  templates, and real Play Integrity/OEM provider smoke are intentionally
  deferred to the next iteration.

## [0.2.0] - 2026-05-17

### Added
- GitHub Packages Maven publishing for
  `io.leonasec:leona-sdk-android:0.2.0`, with GitHub Release AAR + SHA-256
  fallback kept for manual integration.
- Public hosted reporting fallback for the open-source AAR when a Leona hosted
  reporting endpoint and AppKey are configured. The SDK still collects and
  uploads evidence only; customer backends own all final business decisions.
- Backend integration examples for BoxId evidence lookup in Python, Java, Go,
  C, C++, and Node.js, including dry-run signature checks where practical.
- Diagnostic identity fields for `fingerprintSchemaVersion`,
  `fingerprintSource`, `identityAnchorSource`, and
  `canonicalDeviceIdSource`, so support bundles can explain device-id
  stability behavior without exposing raw identifiers.
- Public clean-OEM ledger gate for release checks across mainstream WeTest
  device families without starting paid device sessions.

### Changed
- Tightened public hosted canonical parsing so the SDK only persists explicit
  server canonical ids that match the `L...` format; temporary client `T...`
  ids and raw device ids are not promoted to canonical ids.
- Documented the fingerprint schema v2/v3 compatibility contract. The
  customer-facing stable handle is the server-owned `canonicalDeviceId`; the
  local `fingerprintHash` is a correlation handle and may change across schema
  versions.
- Clarified sample UI and public docs for test-posture facts:
  `runtime.mapping.*` is a runtime mapping fact, while
  `tamper.installer.missing`, ADB, developer options, and debuggable are
  installation/debug posture evidence and are not standalone Root/Hook/Frida
  conclusions.
- Reworked WeTest/cloud-device collection docs to avoid private console
  dependencies and to accept business-backend verdict JSON as the public-safe
  evidence input.

### Fixed
- Clean HUAWEI OEM false-positive telemetry where a bare `generic` substring
  could be rolled up into `rom.generic_aosp_like` /
  `rom.custom_aosp_like`. Generic AOSP evidence now requires AOSP/GSI context.
- Public hosted reporting error classification now distinguishes
  `auth_failed`, `timestamp_skew`, `network_timeout`, and `server_5xx` for
  diagnostics and retry handling.
- Cloud-test direct trigger hardening: the sample receiver requires
  `android.permission.DUMP`, keeping ADB/webshell test control while preventing
  ordinary third-party app spoofing.
- WeTest webshell launch artifacts are redacted before being copied into
  reports.

### Validation notes
- MuMu direct-only identity stability passed initial / clear data / reinstall /
  reboot with matching canonical hashes.
- Android Studio Emulator independent-instance stability passed initial /
  clear data / reinstall / reboot; a cloned AVD produced a distinct stable
  canonical hash from the source AVD.
- WeTest iQOO 7 / Android 14 hosted-reporting smoke passed against the HTTPS
  endpoint without timeout, TLS handshake, auth, timestamp-skew, or server-5xx
  errors.
- GitHub Release AAR + SHA-256 assets were generated by the `v0.2.0` tag
  workflow and verified after download.
- GitHub Packages remote pull passed from an independent Gradle consumer for
  `io.leonasec:leona-sdk-android:0.2.0`, including public transitive
  dependencies.

## [0.1.0] - 2026-05-09

### Added
- First public Android SDK release for hosted Leona API integration.
- Evidence-only client posture: the SDK collects environment evidence and
  returns an opaque BoxId; customer backends own all business decisions.
- Public native runtime evidence for emulator, root, Magisk/KSU, Xposed,
  Frida/runtime mapping, unidbg, APK/resource/signing, and custom-ROM posture
  signals.
- Redacted support-bundle, logcat smoke, posture collection, and WeTest matrix
  scripts for repeatable integration checks.

### Changed
- Stabilized virtual-device fingerprinting so emulator/clone identity no
  longer depends on app-scoped Android ID.
- Hardened cloud config and reporting paths to use hashed identity fields and
  low-trust telemetry for client-provided evidence.
- Updated public docs to reflect the current open-source boundary: Android
  public SDK is open source; hosted backend, private detectors, and risk policy
  remain closed-source for security reasons.

### Fixed
- Clean OEM false positives found during WeTest release-gate runs, including
  vivo `qemu.hw.mainkeys*` navigation-key properties.
- WeTest/cloud-device timestamp skew by using server-time based reporting in
  the hosted/private reporting path.
- Token redaction for webshell/direct cloud-test artifacts.

## [0.1.0-alpha.1] - 2026-04-21

### Added
- Public Android SDK API surface:
  - `Leona.init()`
  - `Leona.sense()`
  - `Leona.senseAsync()`
  - `BoxId`
  - `LeonaConfig`
  - `Honeypot`
  - decoy `quickCheck()`
- Kotlin ↔ JNI ↔ C++ runtime collection path
- Native runtime detection for:
  - Frida / ptrace / trampoline signals
  - emulator signals
  - root / Magisk / KernelSU / Riru traces
  - Xposed / LSPosed / EdXposed traces
  - Unidbg traces
- X25519 + HKDF + AES-GCM + HMAC upload path
- Sample app for local stub mode and real server demo mode
- Minimal demo backend for verdict query demonstration

### Changed
- Sample app now supports Gradle property injection for real demo flow:
  - `LEONA_API_KEY`
  - `LEONA_REPORTING_ENDPOINT`
  - `LEONA_CLOUD_CONFIG_ENDPOINT`
  - `LEONA_DEMO_BACKEND_BASE_URL`
- Project status is now documented as an engineering alpha focused on
  Android SDK ↔ server loop closure

### Known limitations
- Sample app still defaults to local stub mode
- Real Android ↔ server ↔ demo backend acceptance is archived on emulator; real-device archive is still pending
- Public API should still be treated as alpha and may change before beta
