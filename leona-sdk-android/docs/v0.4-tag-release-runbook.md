# Leona Android SDK v0.4 Tag Release Runbook

This runbook is the public-safe sequence for cutting the Android SDK v0.4 tag.
It does not contain credentials, complete BoxIds, raw device identifiers,
provider tokens, signing material, or deployment details.

The SDK remains evidence-only. It reports BoxIds, diagnostics, and redacted
evidence material. Customer backends own final business decisions.

## 1. Pre-Tag Version Alignment

Before creating a real `v0.4.0` tag, update the Android SDK coordinate version:

```properties
VERSION_NAME=0.4.0
```

Then rerun the focused version marker gate and the local release candidate
manifest:

```bash
LEONA_TARGET_RELEASE_VERSION=0.4.0 ./scripts/verify-v0.4-version-bump-plan.py
LEONA_TARGET_RELEASE_VERSION=0.4.0 ./scripts/verify-v0.4-version-bump-dry-run.py

LEONA_REQUIRE_ANDROID_VERSION_MARKERS=1 LEONA_TARGET_RELEASE_VERSION=0.4.0 \
  ./scripts/verify-v0.4-version-markers.sh

LEONA_REQUIRE_PRETAG_READY=1 LEONA_TARGET_RELEASE_VERSION=0.4.0 \
  ./scripts/verify-v0.4-release-candidate-manifest.sh
```

Required result:

- status is `local-pass-with-external-blockers`
- version bump plan status is `pass`
- version bump plan lists required marker files and planned SHA-256 digests
- version bump dry-run status is `pass`
- version bump dry-run proves required markers converge to `0.4.0`
- Android version marker gate status is `pass`
- require pre-tag ready is `1`
- failures are `none`
- `Pre-Tag Blockers` is `none`
- expected AAR asset is `leona-sdk-android-0.4.0.aar`
- expected checksum asset is `leona-sdk-android-0.4.0.aar.sha256`
- public release batch paths and do-not-stage paths are listed
- public release batch staged forbidden paths is `0`
- public release batch paths sha256 and stage command draft paths sha256 match
- public release batch stage command draft syntax is `pass`
- public release batch stage command draft matches public batch is `yes`
- public release batch stage command draft contains do-not-stage paths is `no`
- public release batch stage command dry-run is `pass`
- public release batch stage command dry-run preserves index is `yes`
- public release batch schema status is `pass`
- release evidence pack redaction scan is `pass`
- release evidence pack schema status is `pass`
- release evidence pack lists readiness, archive consumer, publish workflow,
  public commit scope, and public release batch component summaries
- release evidence pack redaction scan includes generated Markdown/JSON and
  git index snapshots, not only component summaries
- release evidence pack schema validates required JSON fields, expected
  components, referenced file digests/byte counts, git index snapshots, and
  Markdown/JSON parity
- release candidate manifest schema validates component paths,
  no-tag/no-publish/no-stage guarantees, public batch fields, index
  preservation, and version blocker semantics
- release evidence pack includes the read-only version bump plan and version
  bump dry-run components
- optional `LEONA_RC_SELF_SCAN_TEST=1` negative test fails on the synthetic
  evidence pack marker while preserving the git index
- release candidate preserves git index is `yes`
- release evidence pack lists before/after git index snapshots
- secret values printed is `no`

Do not create the tag while `VERSION_NAME` still points at an earlier release.

Optionally validate the generated manifest summary before tag review:

```bash
./scripts/verify-v0.4-release-candidate-manifest-schema.py /tmp/<rc-report>/summary.md
```

For one local review command, run:

```bash
./scripts/verify-v0.4-release-candidate-review.sh
```

Optionally validate the review wrapper summary before tag review:

```bash
./scripts/verify-v0.4-release-candidate-review-schema.py /tmp/<review-report>/summary.md
```

For one final local review command, run both the review wrapper and review
schema gate:

```bash
./scripts/verify-v0.4-release-candidate-final-review.sh
./scripts/verify-v0.4-release-candidate-final-review-schema.py /tmp/<final-review-report>/summary.md
```

## 2. Public Commit Scope

Before creating the public commit that will be tagged, verify the staged scope:

```bash
./scripts/verify-v0.4-public-commit-scope.sh
```

Required result for the commit that will receive the tag:

- staged forbidden paths is `0`
- iOS, Web, server, homepage, deployment, policy, private detector, and
  internal-doc paths are not staged
- secret values printed is `no`

For final cleanup, run strict mode and resolve every non-public dirty path
outside the public Android release commit:

```bash
LEONA_REQUIRE_PUBLIC_COMMIT_CLEAN=1 ./scripts/verify-v0.4-public-commit-scope.sh
```

To generate the public Android release batch path list before staging, run:

```bash
./scripts/verify-v0.4-public-release-batch.sh
```

Review `public-release-batch-paths.txt` and `do-not-stage-paths.txt` from the
generated report. Require `stage command draft syntax` to be `pass`, `stage
command draft matches public batch` to be `yes`, `stage command draft contains
do-not-stage paths` to be `no`, `stage command dry-run` to be `pass`,
`stage command dry-run preserves index` to be `yes`, and the public batch /
stage draft SHA-256 digests to match before running any generated
stage-command draft.

## 3. Optional Heavy Local Gates

Run these when the local environment has the required Android toolchain and
network access:

```bash
LEONA_RUN_PUBLIC_GRADLE_GATE=1 ./scripts/verify-v0.4-release-readiness.sh
```

Run post-release consumption only after a release exists:

```bash
LEONA_TARGET_RELEASE_VERSION=0.4.0 \
  LEONA_REQUIRE_POST_RELEASE_CONSUMPTION=1 \
  ./scripts/verify-v0.4-post-release-consumption.sh
```

GitHub Packages consumption requires a read-only packages token from a
non-public secret store.

## 4. Tag And Workflow

Create an annotated tag from the intended public commit:

```bash
git tag -a v0.4.0 -m "Leona Android SDK v0.4.0"
git push origin v0.4.0
```

The public Android workflow must run on the pushed `v*` tag and complete:

- native source sanity
- lint and unit tests
- assemble debug and release AAR
- publish GitHub Release AAR and `.sha256`
- publish GitHub Packages Maven artifact

The workflow must not reference Central, provider, customer, or deployment
secrets.

## 5. Post-Release Consumption Smoke

After the tag workflow completes, verify the public artifact path:

```bash
LEONA_TARGET_RELEASE_VERSION=0.4.0 \
  LEONA_REQUIRE_POST_RELEASE_CONSUMPTION=1 \
  ./scripts/verify-v0.4-post-release-consumption.sh
```

Required result for the fallback path:

- GitHub Release AAR downloads
- `.sha256` downloads
- checksum verification passes

If package-read credentials are available in a non-public environment, rerun
with `LEONA_GITHUB_PACKAGES_TOKEN` to validate GitHub Packages resolution.

## 6. External Blocker Policy

Do not claim these are completed unless real material has been provided and
validated:

- full Android custom ROM / GSI / unlocked / external emulator matrix
- real Play Integrity or OEM attestation provider smoke
- Maven Central publishing
- authenticated hosted ops smoke
- real backend wrapper endpoint smoke

If any are deferred, call them out explicitly in release notes. Synthetic
credentials, dry-runs, or debug providers do not close these blockers.
